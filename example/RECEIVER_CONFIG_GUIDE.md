# 接收器配置详解

本文档详细说明配置界面中「接收器配置」各选项的作用和使用场景。

---

## 📡 接收器概述

系统支持三种日志接收方式：
- **TCP** - 长连接，适合高吞吐量的持续日志流
- **UDP** - 无连接，轻量级，适合大量设备上报
- **HTTP** - REST API，适合应用程序主动推送

---

## 🔧 TCP/UDP 接收器

### 启用 TCP 接收器

```
☑️ 启用TCP接收器 (端口: 9000)
```

| 属性 | 说明 |
|------|------|
| **作用** | 开启 TCP 端口接收日志数据 |
| **默认端口** | 9000 |
| **适用场景** | 长连接、高吞吐量、需要可靠传输 |
| **典型用户** | Nginx、Apache、Filebeat、Logstash |

**使用示例：**
```bash
# Nginx 配置
access_log syslog:server=localhost:9000 main;

# 或使用 nc 测试
echo 'log line' | nc localhost 9000
```

---

### 启用 UDP 接收器

```
☑️ 启用UDP接收器 (端口: 9001)
```

| 属性 | 说明 |
|------|------|
| **作用** | 开启 UDP 端口接收日志数据 |
| **默认端口** | 9001 |
| **适用场景** | 大量设备上报、容忍丢包、低延迟 |
| **典型用户** | Syslog、IoT 设备、Docker 日志 |

**使用示例：**
```bash
# Syslog 发送
echo '<14>Test message' | nc -u localhost 9001

# Docker 日志驱动
docker run --log-driver syslog --log-opt syslog-address=udp://localhost:9001 ...
```

## 🌐 HTTP 接收器

### 启用 HTTP 接收器

```
☑️ 启用HTTP接收器 (端口: 9002)
```

| 属性 | 说明 |
|------|------|
| **作用** | 开启 HTTP API 接收日志数据 |
| **默认端口** | 9002 |
| **适用场景** | 应用程序主动推送、需要认证 |
| **典型用户** | Web 应用、微服务、Serverless |

**使用示例：**
```bash
curl -X POST http://localhost:9002/logs \
  -H "Content-Type: text/plain" \
  -d '127.0.0.1 - - [01/Jan/2024:00:00:00 +0800] "GET /api/test HTTP/1.1" 200 123'
```

---

### 认证 Token

```
认证 Token (留空则不启用认证)
[________________________]
```

| 属性 | 说明 |
|------|------|
| **作用** | 防止未授权访问 HTTP 接口 |
| **留空** | 允许匿名访问（不推荐生产环境） |
| **设置后** | 请求需携带 Token 才能成功 |

**使用方式：**
```bash
# 方式1: Header 传递
curl -X POST http://localhost:9002/logs \
  -H "X-Auth-Token: your-secret-token" \
  -d 'log line'

# 方式2: Query 参数
curl -X POST "http://localhost:9002/logs?token=your-secret-token" \
  -d 'log line'
```

**建议：** 生产环境务必设置 Token，并使用复杂字符串。

---

### IP 白名单

```
IP 白名单 (逗号分隔，留空则不限制)
[________________________]
```

| 属性 | 说明 |
|------|------|
| **作用** | 限制可访问 HTTP 接口的 IP 地址 |
| **留空** | 允许任意 IP 访问 |
| **设置后** | 只有列表中的 IP 能访问 |

**配置示例：**
```
127.0.0.1,192.168.1.100,10.0.0.5
```

**使用场景：**
- 内网部署：只允许内网 IP
- 微服务架构：只允许特定服务 IP
- 安全加固：防止公网暴露

---

### 速率限制

```
速率限制 (每分钟最大请求数，0为不限制)
[0]
```

| 属性 | 说明 |
|------|------|
| **作用** | 防止单个 IP 发送过多请求 |
| **0** | 不限制 |
| **>0** | 每 IP 每分钟最多 N 个请求 |

**配置建议：**

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 开发测试 | 0 | 方便调试 |
| 小规模应用 | 60 | 每秒1个请求 |
| 中等规模 | 600 | 每秒10个请求 |
| 高并发 | 6000+ | 每秒100+请求 |

**超出限制时的响应：**
```json
{"error":"Rate limit exceeded"}
HTTP 429 Too Many Requests
```

## ⚙️ 通用配置

### 缓冲区大小

```
缓冲区大小 (字节)
[8192]
```

| 属性 | 说明 |
|------|------|
| **作用** | 网络读取缓冲区大小 |
| **默认值** | 8192 (8KB) |
| **范围** | 1024 - 65536 |

**调整建议：**

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 小日志行 (<1KB) | 4096 | 节省内存 |
| 标准日志 (1-8KB) | 8192 | 默认即可 |
| 大日志/JSON | 16384+ | 避免截断 |
| 高吞吐量 | 32768+ | 减少读取次数 |

## ⚠️ 常见问题

### 端口冲突
```
Error: listen tcp :9000: bind: address already in use
```
**解决：** 更换端口号，或查找占用进程：`netstat -ano | findstr 9000`

### 无法接收日志
1. 检查接收器是否启用（复选框是否勾选）
2. 检查防火墙是否放行端口
3. 检查端口是否被其他程序占用

### HTTP 返回 401 Unauthorized
- 检查 Token 是否正确
- 检查 Header 名称是否为 `X-Auth-Token`

### HTTP 返回 403 Forbidden
- 检查客户端 IP 是否在白名单中

### HTTP 返回 429 Too Many Requests
- 增加速率限制值
- 或设置为 0 取消限制
