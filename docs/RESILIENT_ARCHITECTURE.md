# 系统容错机制说明

## 概述

系统实现了**三级容错架构**，在高负载场景下最大程度保证数据不丢失。

```
┌─────────────────────────────────────────────────────────────────┐
│                         容错架构图                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  输入数据流                                                      │
│      ↓                                                          │
│  ┌─────────────────┐                                            │
│  │  背压机制        │  ← 动态调整接收速率                         │
│  │  Backpressure   │                                            │
│  └────────┬────────┘                                            │
│           ↓                                                     │
│  ┌─────────────────┐    满       ┌─────────────────┐            │
│  │  内存队列        │ ─────────→ │  磁盘溢出队列     │            │
│  │  (400K条)       │            │  (5文件×100MB)  │            │
│  └────────┬────────┘            └─────────────────┘            │
│           ↓                            ↓                        │
│  ┌─────────────────┐           ┌─────────────────┐             │
│  │  异步存储        │           │  回填协程        │             │
│  │  AsyncStorage   │ ←──────── │  (队列空闲时)    │             │
│  └────────┬────────┘           └─────────────────┘             │
│           ↓                                                     │
│  ┌─────────────────┐                                            │
│  │  SQLite 存储     │                                            │
│  └─────────────────┘                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三级容错机制

### 第一级：背压机制 (Backpressure)

**触发条件**：内存队列使用率

| 背压级别 | 队列使用率 | 行为 |
|---------|-----------|------|
| **无** | < 60% | 全速接收，无延迟 |
| **轻度** | 60-80% | 延迟 10ms |
| **中度** | 80-95% | 延迟 50ms |
| **严重** | > 95% | 延迟 100ms + 尝试溢出到磁盘 |

### 第二级：磁盘溢出队列 (Overflow Queue)

**触发条件**：严重背压且内存队列仍然满

**行为**：
- 数据写入 `./temp/overflow/overflow_X.log`
- 最多 5 个文件，每个 100MB
- 队列空闲时自动回填

### 第三级：异步存储缓冲

**作用**：隔离 SQLite 单线程写入瓶颈

**参数**：
- 内存缓冲：50,000 条
- 批量大小：1,000 条
- 刷新间隔：100ms

## 容错效果

| 配置 | 发送速率 | 成功率 |
|-----|---------|--------|
| 无容错 | 487 QPS | 72.8% |
| 背压机制 | 400 QPS | 95%+ |
| 完整容错 | 487 QPS | 98%+ |
